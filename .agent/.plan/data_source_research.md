# PatentX 数据源获取与评估可行性调研报告

## 1. 核心专利数据源多维度对比

根据调研，获取国内专利（CN）及欧洲专利（EP）并进行特征比对，主要有以下三大核心数据源：

| 维度 | 欧洲专利局官方 API (EPO OPS) | Google Patents BigQuery 公开数据集 | The Lens API (Lens.org) |
| :--- | :--- | :--- | :--- |
| **获取渠道** | [EPO Developer Portal](https://developers.epo.org/) | GCP BigQuery `patents-public-data` | [Lens.org Developers](https://about.lens.org/lens-apis/) |
| **注册流程** | 注册开发者账号，在 "My Apps" 下申请 OPS 产品获取 Consumer Key/Secret。 | 创建 GCP 项目，启用 BigQuery API，（可选）绑定账单。 | 注册 Lens 账户，在个人 Profile 提交 API 接入申请说明用途。 |
| **技术接入** | 中等。OAuth 2.0 认证，返回 XML 格式，需要解析深层节点与处理复杂的“红绿灯”动态限流。 | 低。支持标准 SQL 检索，使用官方客户端 SDK，与 Python、Pandas、AI 流水线无缝集成。 | 低。Header 携带 Bearer Token，返回标准 JSON，支持类似 Elasticsearch 的 DSL 查询。 |
| **价格与限制** | 每周免流量 **4 GB**，超出后需要 **2,800 欧元/年**。高峰期限制极其严格。 | 计算按扫描量收费，**每月前 1 TB 免费**，超出后 $5/TB，存储完全免费。 | 学术/非营利提供 LEAP 免费资助计划；商业用途需要定制商业谈判。 |
| **实时性** | 极高（官方实时数据） | 较低（有数周数据同步延迟） | 较高（有数天延迟） |
| **核心优势** | EP 法律状态、最新审查报告最权威。 | 适合进行全库级别离线特征提取与向量检索库初始化。 | 完美打通了全球专利与学术文献（Non-Patent Literature）的关联。 |

## 2. 宏观实施战略路线

为推进“把国内专利放到 EPO 体系下评估”的目标，建议在以下两条宏观战略路线中进行选择：

### ⚔️ 战略方向一：离线特征库构建 + 大模型语义检索架构（AI 驱动型）
*   **核心逻辑**：
    1. 利用 **Google Patents BigQuery** 导出与国内企业出海领域强相关的 CPC/IPC 分类号下的所有 CN 和 EP 专利文本。
    2. 使用 Embeddings 模型（如 Text-Embedding-3-Large）对提取的专利特征（权利要求、摘要等）进行向量化，并在本地构建高效的向量数据库（如 pgvector 或 Milvus）。
    3. 用户上传国内专利（CN）后，通过向量检索快速召回 Top 50 潜在冲突的欧洲现有技术（Prior Art）。
    4. 将召回的专利交由 L2 层专精 Agent（新颖性/创造性审查员）进行精细化一对一比对，生成诊断看板。
*   **适用场景**：对检索准确度、技术特征深层匹配有高要求，且未来希望沉淀核心 AI 检索资产的项目。

### 🔌 战略方向二：在线实时聚合检索 + 人工干预系统架构（API 驱动型）
*   **核心逻辑**：
    1. 用户在前端 UploadHub 上传国内专利或输入专利号，后端通过 LLM 提取关键特征词（Keywords）及 CPC 分类号。
    2. 通过 **The Lens API** 或 **EPO OPS API** 实时发送结构化检索请求，直接获取欧洲专利局相关的对比文件列表。
    3. 后端收到 API 结果后，实时提取对比专利的权利要求与中文进行流式语义比对。
    4. 若在限流时间窗内未完成或 API 超额，系统自动触发 HITL（Human-in-the-Loop）暂停卡片，提示“系统正在进入高级人工审核队列”，待后台同步后恢复。
*   **适用场景**：希望轻量化起步，不想维护庞大本地数据库，且能快速验证商业闭环的 SaaS 系统。
